# Calendar Policy Smoke Test

대상: `app/src/main/assets/html/calendar.html`  
기준일: 2026-03-05

## 사전 조건
1. 테스트 프로필에 오늘/과거/미래 날짜 TODO가 각각 1개 이상 존재.
2. 오늘 날짜 TODO는 최소 1개가 미완료 상태.

## 체크리스트
1. 과거 날짜 선택 시 수정/삭제 버튼이 비활성화되고 탭 시 정책 안내가 노출된다.
2. 오늘 날짜 선택 시 완료 체크 버튼이 활성화된다.
3. 오늘 날짜에서 완료 체크 시 즉시 체크 상태가 바뀌고 재진입 후에도 유지된다.
4. 미래 날짜 선택 시 완료 체크 버튼이 비활성화된다.
5. TODO 삭제 시 목록에서 제거되고 주간 진행률이 즉시 재계산된다.
6. 날짜를 바꾸면 주간 진행률(`현재 포인트`)이 해당 주 데이터 기준으로 갱신된다.

## 정적 점검(이번 턴)
1. `data-toggle-id` 완료 토글 버튼: PASS
2. 과거 수정/삭제 제한 문구 및 가드: PASS
3. 오늘만 완료 가능 가드: PASS
4. `loadWeeklyProgressForSelectedDate` 주간 실집계 로직: PASS
