## 이번 세션 반영
| 항목 | 결과 |
|---|---|
| `seedSampleData` 성공/실패 신호 | `onSampleSeeded` / `onError`로 분기 처리 |
| 샘플 모드 포인트/명언 덮어쓰기 | `todo_main`에서 API 재덮어쓰기 차단 |
| 샘플 완료 처리 저장 | `localSampleTodos` 완료 상태 영속화 |
| 샘플 기본 프로필 키 | `localSamplePrimaryProfileId` 도입 및 사용 |
| 팀 목록 초기 렌더 | 원격 이미지 의존 제거 + 로컬 선렌더 적용 |
| 단위 테스트(TDD) | `FirestoreManagerTest` 추가 및 통과 |
| 친구 할일 포인트 정합성 | `friend_todo`에서 하드코딩(140/+2p) 제거, 완료 TODO 기준 실집계 적용 |
| 친구 할일 응원 상태 동기화 | `getThumbsUps` 기반 좋아요 수/내 응원 여부 로드 및 토글 후 재동기화 |
| 메인 포인트 덮어쓰기 방지 | `todo_main`에서 날짜별 TODO 집계를 우선하고, 오늘+빈목록에서만 `getPoints` 백필 적용 |
| 캘린더 제한 정책/주간 진행률 | 지난 날짜 수정/삭제 제한, 오늘만 완료 토글, 주간 진행률을 주간 TODO 실집계로 전환 |
| 목표 상세 데이터 바인딩 | `goal_detail`에서 선택 목표 ID/제목으로 데이터 로드해 카테고리/기간/달성률 UI 동기화 |
| 목표 선택 상태 정합성 | `goal_list`에서 ID 없는 카드 선택 시 `selectedGoalId` 초기화로 상세 오진입 방지 |

## 남은 작업 순서
1. `todo_main`/`friend_todo` 하트 애니메이션/포인트 취소 시나리오 실기기 최종 검증.
2. `calendar` 날짜 정책(과거 수정/삭제 제한, 오늘 완료 토글) 실기기 회귀 점검.
3. 프로필 관련 화면(`profile_select`, `profile_manage`, `settings`) 로딩/테마/텍스트 깨짐 회귀 점검.
4. stitch 원본 대비 UI/기능 차이표 업데이트 후 최종 APK 빌드/설치/스모크 테스트.
