## 회귀 위험 체크
| 항목 | 상태 | 메모 |
|---|---|---|
| 하단 네비 5개 메뉴 순서/아이콘 | 양호 | `todo_main`, `calendar`, `team_list`, `goal_list`, `settings` 기준으로 통일 확인 |
| 친구/내 프로필 분기 진입 | 양호 | `profile_select`에서 `isMyAccount` 기준 분기, 친구는 `friend_todo`로 이동 |
| 샘플 데이터 모드 프로필 불일치 | 개선 | `localSamplePrimaryProfileId` 우선 사용 로직 반영(`settings`, `todo_main`, `calendar`) |
| 팀 목록 초기 지연(이미지) | 개선 | 초기 하드코딩 원격 이미지 제거 + 로컬 선렌더 적용 |
| 한글 깨짐 | 주의 | HTML 본문 문자열은 정상. 일부 주석/라벨(개발 텍스트) 깨짐 흔적은 별도 정리 필요 |

## 결론
- 원본 UI/기능의 뼈대(헤더/카드/네비/핵심 인터랙션)는 현재 버전에서 유지되고 있습니다.
- 차이는 대부분 데이터 연동/상태 복원/성능 보강을 위한 확장입니다.
- 사용자 체감 이슈 가능성이 큰 부분(프로필 분기, 포인트 반영, 팝업/네비 동작, 첫 렌더 지연)은 우선 보강되어 회귀 위험이 낮아졌습니다.
